# SF-WhatDreamsCost-ComfyUI

基于 [WhatDreamsCost/WhatDreamsCost-ComfyUI](https://github.com/WhatDreamsCost/WhatDreamsCost-ComfyUI) 二次开发。
基于https://github.com/yg496/CS-WhatDreamsCost-ComfyUI二次开发
感谢原代码作者Jonathan Watkins以及work-fisher和尺素呵等一众大佬

SF 版本核心增强：**自动识别万象分镜脚本视频提示词中的宫格位置标注和时长标注，实现音画时长同步、视频和谐连贯**。

配合 [novel-to-wanxiang-storyboard](https://github.com/rickSF/novel-to-wanxiang-storyboard) 技能使用，可构建"小说 → 万象分镜脚本 → ComfyUI 自动解析 → LTX 视频生成"的完整工作流。

## SF 版本核心增强

### 1. 宫格位置标注自动识别

新增 `prParseGridPositionText` / `parseGridPositionText` 解析函数（Python 后端 + JS 前端双端实现），自动识别视频提示词中的宫格位置标注，将单段连续文字拆分为对应宫格位置的提示词。

**支持两种格式（双兼容）：**

**新格式 V2.4+（推荐）：** 每格独立序号
```
[风格提示词]。第一个镜头，左上，中景，3秒，[描述]。第二个镜头，右上，近景，4秒，[描述]...
```

**旧格式 V2.3（兼容）：** 单序号 + 位置标注
```
[风格提示词]。第一个镜头（对应2x2四宫格从左上至右下区域）：左上，中景，3秒，[描述]。右上，近景，4秒，[描述]...
```

**支持的宫格位置标注：**
- 4宫格（2x2）：左上、右上、左下、右下
- 6宫格（2x3）：左上、中上、右上、左下、中下、右下
- 9宫格（3x3）：左上、中上、右上、左中、中中、右中、左下、中下、右下

### 2. 时长标注自动解析

自动解析视频提示词中每格的时长标注（如"4秒，"），将秒数转换为帧数传递给导演台时间线，实现非平均时长分配。

**时长分配原则：**
- 有台词的格时长长一些（保证台词能说完）
- 近景和特写时长长一些（展示表情细节）
- 中景时长短一些（仅交代位置）
- 所有格时长之和等于锁定总时长（4宫格=12秒，6宫格=18秒，9宫格=27秒）

### 3. 风格提示词保留

视频提示词开头的风格提示词会被保留到第一个镜头的提示词中，不会被切掉，确保生成的第一张图片也带有完整风格信息。

### 4. 万象分镜脚本兼容

完全兼容万象软件的视频提示词格式：
- 视频提示词是一段连续文字（万象软件可识别）
- 宫格位置标注和时长标注内联在文字中
- 万象软件读取整段文字作为提示词，不影响识别
- ComfyUI 插件自动解析其中的宫格位置和时长信息

## 视频提示词格式示例

### 新格式 V2.4+（推荐）

```
视频提示词:电影感真人摄影，现代都市剧照，便服日常装，都市室内与街市场景，自然光与暖黄室内光，浅景深，35mm镜头质感，无文字无水印无边框。第一个镜头，左上，中景，3秒，李强穿蓝色旧T恤坐在单人床边，一手握着彩票一手拿着手机，台灯暖黄光照亮简陋出租屋，镜头以定镜开场静静记录他的背影和侧脸，纸张沙沙声让空间显得安静。第二个镜头，右上，近景，4秒，李强的手指移向彩票，指向上面的数字，瞳孔微缩，表情从专注转为震惊，镜头缓慢推近聚焦他的眼睛和微张的嘴巴。[李强用标准普通话（颤抖着确认）：全对上了，一等奖，五百万，我李强也有今天。]第三个镜头，左下，特写，3秒，彩票特写置于掌心，红蓝双色印刷的数字清晰可见。第四个镜头，右下，近景，2秒，李强猛地抬头，嘴巴张开，右手缓缓握拳。
```

**格式说明：**
1. 开头是风格提示词（会保留到第一个镜头的提示词中）
2. 每格用独立序号标注："第一个镜头，""第二个镜头，"...
3. 序号后紧跟宫格位置标注，用逗号分隔："第一个镜头，左上，"
4. 每格描述格式："[宫格位置]，[景别]，[时长]秒，[描述内容]"
5. 各格之间用句号"."分隔
6. 整段是一段连续文字，不能有空行或换行
7. 台词用方括号嵌入：`[角色名用标准普通话（语气状态）：台词内容。]`

### 旧格式 V2.3（兼容）

```
视频提示词:电影感真人摄影，现代都市剧照。第一个镜头（对应2x2四宫格从左上至右下区域）：左上，中景，3秒，李强坐在床边翻看彩票。右上，近景，4秒，李强低头看手机屏幕。左下，特写，3秒，彩票特写。右下，近景，2秒，李强握拳。
```

## 基本流程

1. 准备一张四宫格、六宫格或九宫格参考图
2. 将参考图连接到 `宫格图像` 输入
3. 将万象分镜脚本生成的视频提示词连接到 `GPT 分镜文本` 输入
4. 选择 `宫格模式`，例如 `3x3 九宫格`
5. 选择 `分镜比例`，例如 `16:9 横屏` 或 `9:16 竖屏`
6. 根据宫格边框情况调整 `白边裁剪强度`
7. 点击 `自动填充宫格`，节点会自动识别视频提示词中的宫格位置标注和时长标注，生成对应数量的可编辑分镜块，并按时长分配每段帧数
8. 在导演台前端检查并手动修改每段分镜文本、时长或顺序
9. 将 `引导数据` 接到 `SF-LTX 导演台引导`，继续连接 LTX 采样和解码节点

## 宫格模式

拆分顺序始终为从左到右、从上到下。

| 模式 | 分镜数量 | 顺序 |
| --- | ---: | --- |
| `2x2 四宫格` | 4 | 左上、右上、左下、右下 |
| `3x2 六宫格` | 6 | 左上、中上、右上、左下、中下、右下 |
| `3x3 九宫格` | 9 | 左上、中上、右上、左中、中中、右中、左下、中下、右下 |

## 分镜比例

`分镜比例` 控制每个宫格单元拆出来之后的最终画面比例。

| 选项 | 说明 |
| --- | --- |
| `自动 / 保持单格比例` | 保持原始宫格单元比例，最稳妥 |
| `16:9 横屏` | 将每个分镜居中裁切为横屏比例 |
| `9:16 竖屏` | 将每个分镜居中裁切为竖屏比例 |
| `1:1 方图` | 将每个分镜居中裁切为正方形 |

## 安装

将仓库放入 ComfyUI 的 `custom_nodes` 目录，然后重启 ComfyUI：

```text
ComfyUI/custom_nodes/SF-WhatDreamsCost-ComfyUI
```

或通过 ComfyUI Manager 搜索 `SF-WhatDreamsCost` 安装。

## 与 novel-to-wanxiang-storyboard 技能配合使用

1. 使用 `novel-to-wanxiang-storyboard` 技能（V2.4+）生成分镜脚本
2. 将生成的视频提示词复制到 ComfyUI 的 `SF-LTX 宫格导演台` 节点的 `GPT 分镜文本` 输入框
3. 插件自动识别宫格位置标注和时长标注
4. 自动按时长分配每段帧数，确保音画同步
5. 风格提示词自动保留到第一个镜头的提示词中

## 使用注意事项

- `每段帧数` 输入框**留空**，插件会自动用视频提示词中的时长标注分配帧数；若手动填写则覆盖自动分配
- `总帧数` 应与锁定总时长匹配：4宫格12秒=288帧(24fps)、6宫格18秒=432帧、9宫格27秒=648帧
- 视频提示词必须是**一段连续文字**，宫格位置标注（左上/中上/...）和时长标注（N秒，）内联其中，不能换行分段
- 视频提示词开头的风格提示词会自动保留到第一个镜头的提示词中
- 如果同时安装了原始版 `WhatDreamsCost-ComfyUI`，两者节点 ID 不冲突（SF版全带 `SF-` 前缀），可安全并存

## 技术架构

### 双端解析（Python 后端 + JS 前端）

ComfyUI 自定义节点有 Python 后端和 JS 前端两部分，SF 版本在两端都实现了宫格解析：

| 端 | 文件 | 函数 | 作用 |
| --- | --- | --- | --- |
| Python 后端 | `ltx_auto_director.py` | `_segments_from_grid_text` | 后端解析，生成时间线 |
| Python 后端 | `ltx_sixgrid_director.py` | `_segments_from_grid_text` | 宫格导演台后端解析 |
| JS 前端 | `js/ltx_auto_director.js` | `parseGridPositionText` | 自动导演台前端解析 |
| JS 前端 | `js/ltx_director.js` | `prParseGridPositionText` | 宫格导演台前端解析 |

**关键设计：**
- 两端解析逻辑保持一致，确保前端预览与后端执行结果匹配
- 解析顺序：先提取时长标记，再拼接风格提示词（避免风格提示词把时长推到查找范围之外）
- 双格式兼容：先尝试 V2.4 新格式（按"第N个镜头，"切分），失败再走 V2.3 旧格式（按宫格位置标注切分）

### 强制重建时间线

当检测到视频提示词包含宫格位置标注时，`SF-LTX 宫格导演台` 会**强制重建时间线**，忽略前端残留的 `timeline_data`，确保宫格解析逻辑被触发。

### v2.1.5（2026-06-21）

**新增**：剥离"正面词："前缀 + 更新示例工作流。

- **新增**：自动剥离视频提示词开头的"正面词："前缀（用户从 ComfyUI 正面词输入框粘贴时可能带上），支持中文冒号和英文冒号。
- **更新**：示例工作流替换为"万象四宫格导演台横屏1280分辨率12秒（本地版-sf导演台）"，删除旧的英文示例工作流。
- **新增**：测试文件 `test_positive_prefix.js`（17 个用例）。

### v2.1.4（2026-06-20）

**修复**：风格提示词被切掉的问题。

- **问题现象**：视频提示词开头的风格提示词在解析时被丢弃，没有加入到第一个镜头的提示词中。
- **根因**：解析函数按"第N个镜头，"切分时，第一个镜头的 segment 从标记之后开始，标记之前的风格提示词被丢弃。
- **修复**：提取风格提示词（第一个"第N个镜头，"之前的内容），拼接到第一个镜头的 prompt 前面。
- **关键设计**：先提取时长标记，再拼接风格提示词（避免风格提示词把"N秒"推到 30 字符查找范围之外）。
- **新增**：`joinStylePrefix` 辅助函数，智能处理句号分隔。

### v2.1.3（2026-06-20）

**修复**：`ltx_director.js` 独立解析逻辑未同步的问题。

- **问题现象**：用户点击"自动填充宫格"按钮后仍然无法正确拆分。
- **根因**：ComfyUI 有两个导演台节点，各自有独立的 JS 前端文件和解析函数。之前只修复了 `ltx_auto_director.js` 的 `parsePrompts`，但 `ltx_director.js` 的 `prParsePrompts`（带 `pr` 前缀，PromptRelay 命名空间）完全没有宫格解析。
- **修复**：`ltx_director.js` 新增 `prParseGridPositionText` 函数，修改 `prParsePrompts` 和 `prResolveSegmentLengths` 支持宫格解析和时长提取。

### v2.1.2（2026-06-20）

**修复**：JS 前端缺少宫格位置标注解析。

- **问题现象**：前端显示只有 1 个 segment，所有提示词挤在第一个镜头。
- **根因**：JS 前端 `parsePrompts` 函数没有宫格位置标注解析逻辑，只按"第N个镜头"前缀切分，而当前格式只有 1 个"第一个镜头" → 只拆出 1 个 prompt。
- **修复**：`js/ltx_auto_director.js` 新增 `parseGridPositionText` 函数，`parsePrompts` 在 auto 模式下优先调用它。

### v2.1.0（2026-06-20）

**修复**：前端残留 `timeline_data` 导致宫格解析被跳过的严重问题。

- **问题现象**：所有提示词挤在第一个镜头、时长被平均分配。
- **根因**：ComfyUI 前端时间线编辑器默认会塞入 segments，使后端 `if not timeline["segments"]` 条件为假，`_build_default_timeline`（宫格解析核心）永远不被触发。
- **修复**：`SF-LTX 宫格导演台` 检测到视频提示词包含宫格位置标注时，**强制重建时间线**，忽略前端残留的 `timeline_data`。
- **优化**：`SF-LTX 自动导演台` 检测到宫格格式时用解析出的分镜数量覆盖 `segment_count`。

### v2.0.0

- 从 CS- 前缀改为 SF- 前缀，重命名为 SF-WhatDreamsCost-ComfyUI
- 新增 `_segments_from_grid_text` 解析函数，识别宫格位置标注和时长标注
